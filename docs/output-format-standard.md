# エージェント出力フォーマット標準

## 目的

全エージェントの出力を統一し、Kennyおよびチームが素早く読み取れるようにする。

## 共通ルール（全エージェント適用）

1. **見出しは`##`から** — `#`はファイルタイトル用
2. **1セクション最大10行** — 長文禁止
3. **表形式を優先** — 箇条書きの羅列より読みやすい
4. **アクションは`- [ ]`** — チェックボックスで出す
5. **数字は具体的に** — 「多い」→「43件」、「高い」→「23%」
6. **NGワード不使用** — CLAUDE.md定義のNGワードは絶対使わない

## 標準出力テンプレート

全エージェントは以下の構造をベースに、自分の役割に合わせてカスタマイズする。

```markdown
## [動詞]: [テーマ]

### サマリー（3行以内）
[結論を最初に。何を・なぜ・いくらで]

### 分析 / 根拠
| 項目 | 内容 |
|------|------|
| ... | ... |

### 推奨アクション
- [ ] [具体的タスク（誰が・何を・いつまで）]
- [ ] [...]

### リスク / 注意点
[1〜3行]

### 参照元
[使用したdocs/ファイル名]
```

## 品質チェックリスト

出力完了時に自己チェック:

- [ ] NGワードを使っていない
- [ ] 数字は具体的（「多い」「高い」を使っていない）
- [ ] アクションアイテムに「誰が」「いつまで」が入っている
- [ ] 1セクション10行以内
- [ ] 参照元ドキュメントを明記している
- [ ] ペルソナに合ったトーンになっている
