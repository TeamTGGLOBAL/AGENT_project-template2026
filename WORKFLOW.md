## Case ライフサイクル：サブモジュールの分離判断

Caseは起点となった展示会・プロジェクト（親リポジトリ）のサブモジュールとして管理するが、以下の条件を満たしたとき**親から切り離して独立リポジトリに昇格させる**。

| 判断軸 | 分離を検討するシグナル |
|---|---|
| 時間的距離 | 案件が展示会終了から6ヶ月以上経過し、起点との文脈的つながりが薄れた |
| 取引規模 | 定期発注が確立し、展示会という文脈なしに取引相手が成立している |
| チーム・担当 | 別担当者がこの案件を引き継ぐ、または専任化する |
| 親リポジトリの役割終了 | 展示会プロジェクト自体がアーカイブ・クローズされる |

**分離の手順**（概要）:
1. Case○○○ リポジトリのリモートURLはすでに独立しているため、git操作は不要
2. 親リポジトリの `.gitmodules` からエントリを削除
3. Case○○○ の `CLAUDE.md` の「情報源」セクションにある親リポジトリへのパス参照を更新
4. `MEMORY.md` に分離日と理由を記録してコミット

> **原則**: 「迷ったら残す」。分離は取引関係が自立したことの証明であり、タイミングを急ぐ必要はない。
