# ヒアリング設計

<!--
このドキュメントの目的:
「誰に・何を・どう聞くか」を事前に設計する。

ヒアリングは「貴重な信頼を消費する行為」。
聞くべきことを整理せずに接触すると、同じことを何度も聞くことになり信頼を失う。

一つの情報源につき一つのファイルを作る。
複数の情報源がある場合はファイル名で区別する。
例: phase1-hearing-kashu.md / phase1-hearing-nakamura.md
-->

---

## 聞き方の設計

<!--
「何を聞くか」と同じくらい「どう聞くか」が重要。

設計すべき内容:
1. 場作り: 会話を始めやすくする準備（ギフト・近況確認・共通の話題）
2. 話の導き方: 直接的な質問を避けて自然な会話の流れで引き出す
3. NG: この情報源の立場を危うくするような聞き方
4. 信頼維持: ヒアリング後に信頼関係を保つための行動

実績があれば記録する。
例: 「スタバのギフトカードを送ると回答してもらいやすい（過去実績あり）」
