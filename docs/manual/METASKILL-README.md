# PRINCIPAL_MetaSkill-AI2026

このリポジトリは、書籍『メタスキル』をOCRでテキスト化し、その内容を理解して、仕事で使える再現性のあるスキルを注入するためのモジュールや資産をまとめたものです。

## 本書の概要

本書は、**AI時代に必要な「メタスキル」** を定義し、これから働く上でAIをうまく活用するための思考法と実践的な手法を提示しています。

### メタスキルとは
従来の「努力」や「実行力」の価値が変わる時代において、**AIを使って価値を生み出す「構造」を設計する視点** そのものがメタスキルです。単なる作業代行ではなく、「思考の拡張装置」としてAIを活用し、自分を最も活かせる「生き方の構造」を作るための具体的な道筋を提唱しています。

## 3つの戦略スキル（Claude Code Skills）

本プロジェクトでは、3つの戦略モデルをClaude Code Skillとして実装しています。
各スキルを呼び出すことで、その流派の思考パターンで問題を分析できます。

### `/fukatsu-strategy` - 深津流戦略エージェント

**コンセプト**: 「死なない」ための構造化とリスク管理

**4つのメタスキル**:
- 事前検死 〜失敗の先回り〜
- ダメージ上限設定
- AI日報 IDEAL
- 面制圧

**思考の特徴**:
- リスクを先に可視化し、「死なない」構造を作る
- 最悪のケースから逆算して対策を打つ
- 不確実性を「確定ダメージ」に変換
- 継続的改善で高速フィードバックループを回す

**使い方**:
```bash
/fukatsu-strategy 新規事業を立ち上げたいのですが、どう考えるべきですか？
```

---

### `/kensu-strategy` - けんすう流戦略エージェント

**コンセプト**: ゲームのルールをずらして、人と違う戦略をとる

**4つのメタスキル**:
- 構造を捉える問いの立て方
- KPIハック
- バイアスを破る極端な質問術
- 意地悪な質問を先回りする反論シミュレーション

**思考の特徴**:
- 一段上からゲームの構造を理解する
- 評価軸をずらして、自分だけの土俵を見つける
- 非対称戦略で、AIや大企業が真似できない勝ち方を作る
- 極端な問いで思考を拡張する

**使い方**:
```bash
/kensu-strategy SNSでフォロワーを増やしたいのですが、どうすべきですか？
```

---

### `/obara-strategy` - 尾原流戦略エージェント

**コンセプト**: 自分の資産を高速回転させて圧倒的なレバレッジをかける

**4つのメタスキル**:
- Z軸リサーチ
- 52か国語スキャン
- 自分モジュール化
- AIを思考のスパーリングパートナーに

**思考の特徴**:
- フライホイール型の仕組みで資産を循環させる
- レバレッジポイントを見つけて、少ない労力で大きな成果
- 多角的な情報収集で独自の洞察を得る
- AIで実行速度を10倍にする高速実験

**使い方**:
```bash
/obara-strategy ブログを書いているのですが、なかなか成果が出ません
```

---

### 3つのスキルの使い分け

同じ問題でも、3つの異なる視点から戦略的アドバイスを受けられます:

- **不安・リスクが大きいとき** → `/fukatsu-strategy` で「死なない」構造を作る
- **競争が激しくて勝てないとき** → `/kensu-strategy` でゲームをずらす
- **成長を加速させたいとき** → `/obara-strategy` でレバレッジをかける

推奨される組み合わせ:
1. まず `/fukatsu-strategy` で土台を固める（生存の土台）
2. `/obara-strategy` でレバレッジをかけて成長加速（増幅の仕組み）
3. 行き詰まったら `/kensu-strategy` で別の土俵を探す（納得の軸）

## 他のプロジェクトへの適用方法

このリポジトリの3つの戦略スキルは、**他のプロジェクトやリポジトリと組み合わせて使う**ことで真価を発揮します。

### 方法1: スキルのインストール（簡易版・推奨）

このリポジトリ全体をクローンせず、**3つのスキルファイルだけを他のプロジェクトにインストール**できます。

#### 手動インストール

```bash
# インストール先のプロジェクトに移動
cd /path/to/your-project

# .claude/skills ディレクトリを作成（なければ）
mkdir -p .claude/skills

# 3つのスキルファイルをコピー
cp /path/to/PRINCIPAL_MetaSkill-AI2026/.claude/skills/fukatsu-strategy.md .claude/skills/
cp /path/to/PRINCIPAL_MetaSkill-AI2026/.claude/skills/kensu-strategy.md .claude/skills/
cp /path/to/PRINCIPAL_MetaSkill-AI2026/.claude/skills/obara-strategy.md .claude/skills/
```

**Windowsの場合（PowerShell）**:
```powershell
# インストール先のプロジェクトに移動
cd D:\path\to\your-project

# .claude/skills ディレクトリを作成
New-Item -ItemType Directory -Force -Path .claude\skills

# 3つのスキルファイルをコピー
Copy-Item D:\Warp_Repository\PRINCIPAL_MetaSkill-AI2026\.claude\skills\*.md .claude\skills\
```

#### ワンライナーインストール（bash）

```bash
# プロジェクトルートで実行
mkdir -p .claude/skills && cp /path/to/PRINCIPAL_MetaSkill-AI2026/.claude/skills/*.md .claude/skills/
```

#### ワンライナーインストール（PowerShell）

```powershell
# プロジェクトルートで実行
New-Item -ItemType Directory -Force -Path .claude\skills; Copy-Item D:\Warp_Repository\PRINCIPAL_MetaSkill-AI2026\.claude\skills\*.md .claude\skills\
```

#### インストール後の使い方

インストールが完了したら、そのプロジェクト内で直接スキルを呼び出せます：

```bash
# プロジェクトのコードを分析
/fukatsu-strategy このプロジェクトのリスクを洗い出してください

# プロジェクトの戦略を考える
/kensu-strategy 競合との差別化ポイントを提案してください

# プロジェクトをレバレッジ
/obara-strategy この資産をどう循環させるか教えてください
```

**メリット**:
- 軽量：スキルファイルだけなので数KB程度
- 独立：このリポジトリ全体をクローンする必要がない
- プロジェクト固有：各プロジェクトでスキルをカスタマイズ可能
- バージョン管理：プロジェクトのgitで管理できる

#### .gitignore に追加する場合

スキルをプロジェクトに含めたくない場合：

```bash
# .gitignore に追加
echo ".claude/skills/*-strategy.md" >> .gitignore
```

逆に、チーム全体で使いたい場合はコミットしてプッシュ：

```bash
git add .claude/skills/
git commit -m "Add 3 strategy skills (深津流/けんすう流/尾原流)"
git push
```

### 方法2: リポジトリメンション（フル機能版）

このリポジトリ全体を参照しながら、他のプロジェクトを分析する方法です。

1. **分析したいリポジトリをクローン**
   ```bash
   # 例: 分析したいプロジェクトをクローン
   ```

2. **Claude Codeでプロジェクトをメンション**
   ```bash
   # @でローカルパスをメンションして、プロジェクトを学習させる
   @D:\path\to\your-project このプロジェクトの構造を教えてください
   ```

3. **戦略スキルで分析**
   ```bash
   # 深津流で分析
   /fukatsu-strategy このプロジェクトのリスクを洗い出してください
   
   # けんすう流で分析
   /kensu-strategy このプロジェクトの差別化戦略を考えてください
   
   # 尾原流で分析
   /obara-strategy このプロジェクトをレバレッジする方法を提案してください
   ```

### 2つの方法の比較

| 項目 | 方法1: スキルのインストール | 方法2: リポジトリメンション |
|------|---------------------------|--------------------------|
| **セットアップ** | スキルファイルをコピー（3ファイル） | リポジトリ全体をクローン |
| **容量** | 数KB | 全体（OCRテキスト含む） |
| **使いやすさ** | ⭐⭐⭐ 簡単 | ⭐⭐ やや複雑 |
| **独立性** | ✅ プロジェクトだけで完結 | ❌ このリポジトリが必要 |
| **カスタマイズ** | ✅ プロジェクト固有に編集可能 | ❌ 元のスキルを参照 |
| **チーム共有** | ✅ プロジェクトのgitで管理 | ❌ 各自がクローン必要 |
| **更新** | ❌ 手動で再コピー | ✅ git pull で最新版 |
| **本の内容参照** | ❌ スキルのみ | ✅ OCRテキスト・解説も参照可能 |

**推奨される使い分け**:
- **個人プロジェクト・チーム開発**: 方法1（インストール）
  - 軽量で、プロジェクトに組み込みやすい
  - チーム全員がスキルを共有できる
- **深い学習・本の内容を参照したい**: 方法2（メンション）
  - OCRテキストや詳細な解説も参照できる
  - メタスキルの理論的背景を理解しながら使える

**両方使う**のもおすすめ：
```bash
# 方法1でスキルをインストール
cd my-project
cp /path/to/PRINCIPAL_MetaSkill-AI2026/.claude/skills/*.md .claude/skills/

# 方法2で詳細な情報を参照

# インストールしたスキルで分析
/fukatsu-strategy このプロジェクトのリスクを分析してください
```
