## プロセス分離（セキュリティ設計）

各ユーザーセッションごとに独立した MCP サーバープロセスを起動することで、
プロセスレベルでユーザーデータを分離しています。

```
セッション A (user@example.com)
  └── MCPプロセス A  [TOKEN_A]  ── GitHubユーザーAのリポジトリのみアクセス可

セッション B (dev@company.com)
  └── MCPプロセス B  [TOKEN_B]  ── GitHubユーザーBのリポジトリのみアクセス可
```

仮にアプリケーションにバグがあっても、別プロセスのトークンが混入する経路がなく、
コンタミネーションリスクを構造的に排除できます。

## セッション管理

```
セッション作成
  │  setSession(id, token)   → メモリに保存（DB 書き込みなし）
  │  startMcpServer(id, token) → 子プロセス起動

アクセスのたびに
  │  getToken(id)            → 最終アクセス時刻を更新

30 分間アクセスなし
  │  cleanupExpiredSessions() → トークン破棄 + プロセス停止

セッション終了（DELETE /session）
  │  stopMcpServer(id)       → SIGTERM → 3秒後 SIGKILL
  │  deleteSession(id)       → トークン破棄
```
