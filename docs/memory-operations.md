# Long-Term Memory 運用ルール

## 概要

Claude Codeのメモリシステム（`~/.claude/projects/<project>/memory/`）を活用し、会話をまたいでコンテキストを蓄積する。

## メモリの種類と使い分け

| 種類 | 用途 | 例 |
|------|------|-----|
| user | ユーザーの役割・好み・スキル | Kennyの作業スタイル、専門知識 |
| feedback | 作業指示のフィードバック | 「止まらず進めて」「リスト見せて」 |
| project | 進行中の作業・目標・期限 | タスク一覧、FY目標、市場計画 |
| reference | 外部リソースの参照先 | Linear、Grafana、Slack等の場所 |

## 書かないもの

- コードから読み取れるパターン・構造
- git logで分かる変更履歴
- CLAUDE.mdに既に書いてある情報
- 一時的なデバッグ情報

## ファイル命名規則

```
{type}-{topic}.md
例: user-kenny.md, project-task-list.md, feedback-work-style.md
```

## MEMORY.mdの管理

- 200行以下に収める
- 各エントリは1行、150文字以内
- `- [タイトル](ファイル名) — 一行フック` の形式

## 更新タイミング

- ユーザーが「覚えて」と言ったとき → 即座に保存
- 作業スタイルの修正指示を受けたとき → feedback として保存
- プロジェクトの状況が変わったとき → project を更新
- 外部リソースの場所を教わったとき → reference として保存

## 読み出しタイミング

- 会話の冒頭で「前回の続き」「覚えてる？」と聞かれたとき
- 作業開始前にタスクリストを確認するとき
- ユーザーの好みに合わせた応答が必要なとき

## 注意事項

- メモリは「その時点の事実」であり、現在も正しいとは限らない
- ファイルパスや関数名を含むメモリは、使用前に実在を確認する
- 重複を避ける: 新規作成前に既存メモリを検索する
